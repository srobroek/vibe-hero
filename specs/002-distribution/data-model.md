This spec's "entities" are packaging/release artifacts, not runtime data. Each is a file or manifest with a defined shape and ownership (hand-authored vs generated). No spec-001 runtime data model changes.

## CLI dispatcher — `packages/server/src/cli/index.ts` (new) → `dist/cli/index.js`

The `bin` target. A thin dispatcher over existing `main()`s:
- argv has no subcommand or `mcp` → import + run the server bootstrap (`src/index.ts` `main()`).
- argv `get-offer [--session --tool]` → import + run the offer CLI (`src/cli/getOffer.ts` `main()`).
- unknown subcommand → stderr usage + nonzero exit.

No business logic; routing only. (`src/index.ts` and `src/cli/getOffer.ts` already export runnable `main()`s.)

## Claude Code marketplace — `.claude-plugin/marketplace.json` (new, generated)

`{ name, owner: {name,url}, plugins: [{ name: "vibe-hero", description, version, tags, source: "." or "./packages/<plugin>" }] }`. Generated by `apm pack` from `apm.yml`; committed; staleness-gated (FR-010/015). vibe-hero's repo IS a remote marketplace (`apm marketplace add srobroek/vibe-hero`).

## Cross-publish entry — in `srobroek/agentic-packages` marketplace (fast-follow, FR-009)

A PackageEntry added to agentic-packages' marketplace with a **remote git source** (APM core first-class):
```yaml
- name: vibe-hero
  source: srobroek/vibe-hero    # remote git (default host github.com); is_local=false
  ref: vibe-hero-v<version>     # tag/branch/sha
  category: ..., tags: [...]
```
No stub, no APM/npm dependency, no source copy. **Blocker**: agentic-packages' local generators (`build_inventory.py`/`render-docs.py`) currently overwrite the packages block from local dirs only and would drop this external entry (+ fail the staleness gate); they must be refactored to preserve external-source entries first. Hence fast-follow, not v1.

## Plugin manifest — `.claude-plugin/plugin.json` (new, generated)

Generated identity manifest (name/version/description/author/license + `skills` path). MUST NOT carry `mcpServers` (MCP is declared in `.mcp.json` only — a duplicate same-name registration in plugin.json conflicts with the `.mcp.json` entry and breaks the plugin). MUST NOT carry name-only `{name}` deps (the standalone plugin has no first-party APM deps, so this is naturally avoided). FR-008/010.

## MCP manifest — `.mcp.json` (new, generated at plugin root)

`{ "mcpServers": { "vibe-hero": { "command": "npx", "args": ["-y", "@vibe-hero/server"] } } }`. Generated from `apm.yml` `dependencies.mcp`. Floating latest (no version) per FR-012. Auto-applied on plugin install → MCP server available with no manual config (FR-006/008).

## Hook manifest — `.apm/hooks/vibe-hero-claude-hooks.json` (new) → generated `hooks/hooks.json`

```json
{ "hooks": { "Stop": [ { "hooks": [
  { "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/hooks/claude-code/stop-offer.sh" } ] } ] } }
```
Uses `${CLAUDE_PLUGIN_ROOT}` — the token the native Claude Code plugin loader substitutes for the plugin's install path in hook `command` strings (every native Claude plugin uses it). The `.apm/hooks/` manifest is the source; the committed `hooks/hooks.json` is what Claude Code discovers and must ship via the marketplace `source`. (`${PLUGIN_ROOT}` is the APM-only token and stays unexpanded under `claude plugin install`, so it must NOT be used here.) Auto-registers on install (FR-007), retiring the manual README.

## Stop hook script — `hooks/claude-code/stop-offer.sh` (REWRITTEN, agent-mediated — FR-011)

A tiny PURE-SHELL script that **spawns nothing**. It:
- reads the Stop payload, respects the `stop_hook_active` loop guard,
- prints `{"hookSpecificOutput":{"hookEventName":"Stop","additionalContext":"<nudge>"}}` where the nudge tells the agent to call the `get_offer` MCP tool (the agent already holds the live MCP connection),
- exits 0 always (never blocks the user).
No `npx`, no `node`, no `get-offer` CLI call, no plugin-local `dist` reference. The old `node dist/cli/getOffer.js` invocation is removed. (The nudge only acts if the agent is still in its loop; the quiz-skill steering reinforces end-of-work offers as a backstop.)
